# Contributing to Script-Bucket

Thanks for contributing! Script-bucket is the community registry for [scriptix](https://github.com/yigitkonur/scriptix) scripts.

## Adding a New Script

### 1. Create the script file

```bash
# Option A: Use scriptix to scaffold
npx scriptix create my-script-name

# Option B: Copy the template
cp scripts/_template.sh scripts/<category>/my-script-name.sh
```

### 2. Write your script

Follow the [CLAUDE.md](./CLAUDE.md) authoring guide. Key requirements:

- Add all `@-directive` metadata at the top
- Use `set -euo pipefail`
- Clean up temp files with `trap`
- Send status messages to stderr (`>&2`)
- Output data only to stdout

### 3. Test locally

```bash
npx scriptix --local ./scripts/<category>/my-script-name.sh "test arg"
```

### 4. Submit a Pull Request

Place your script in `scripts/<category>/my-script-name.sh` where `<category>` matches your `@category` directive.

The `manifest.json` is auto-generated by CI — do not edit it manually.

## Categories

| Category | Description |
|----------|-------------|
| `youtube` | YouTube data extraction and search |
| `github` | GitHub repository and PR tooling |
| `web` | Web scraping, HTTP, URL processing |
| `text` | Text processing and transformation |
| `data` | Data analysis, CSV, JSON manipulation |
| `api` | Generic API integrations |
| `devops` | CI/CD, Docker, infrastructure |
| `media` | Audio, video, image processing |
| `social` | Social media platforms |

Need a new category? Just create the directory and use it in your `@category` directive.

## Quality Guidelines

- Scripts should do **one thing well**
- Prefer common tools (`jq`, `curl`, `grep`) over niche dependencies
- Handle errors gracefully — don't silently fail
- Use descriptive `@arg` descriptions
- Include at least one `@example`
- Test on both macOS and Linux when possible
